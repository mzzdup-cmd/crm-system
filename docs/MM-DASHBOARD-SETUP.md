# MM Dashboard — пошаговая настройка поверх CRM Sync

## Быстрый старт (no-code) — рекомендуется

**Один скрипт создаёт всё автоматически:** листы, формулы, QUERY, оформление, график.

1. Откройте таблицу **MM Dashboard** (лист `Data` с `IMPORTRANGE` уже настроен)
2. **Extensions → Apps Script**
3. Скопируйте **весь файл** из репозитория:  
   [`scripts/google-apps-script/MmDashboardSetup.gs`](../scripts/google-apps-script/MmDashboardSetup.gs)
4. Вверху файла при необходимости укажите `CRM_SYNC_ID` (или оставьте `''` — ID возьмётся из `Data!A1`)
5. Выберите функцию **`setupMmDashboard`** → **Run** → разрешите доступ
6. Вернитесь в таблицу → `Data!A1` → **«Разрешить доступ»** для IMPORTRANGE (если #REF!)
7. Меню **MM Dashboard → Создать / обновить MM** — для повторной пересборки

**CRM Sync не затрагивается.** Скрипт работает только в MM Dashboard.

---

**Цель (ручная настройка — ниже по документу):** визуально повторить реальную MM-таблицу…

**Источник данных:** только 17 колонок из nightly sync:

| Колонка | Поле | Используется в MM |
|---------|------|-------------------|
| A | Дата | периоды (вчера / сегодня / месяц) |
| B | Тип сделки | Новые, ББ, Рассылка, Апсэйл, Доплаты |
| E | Сумма | выручка, средний чек |
| F | Бюджет | средний чек по бюджету |
| J | Откуда | traffic (источник) |
| O | Тариф | Базовый / VIP |
| Q | Менеджер | все сводки по менеджерам |

> **Локаль Google Sheets:** ниже формулы для **русской локали** (`;` как разделитель).  
> Если у вас английская локаль — замените `;` на `,`.

## 0. Что уже должно работать

- Таблица **CRM Sync** с листами `Sync` и `_Meta`
- Nightly sync заполняет `Sync!A:Q`
- Таблица **MM Dashboard** создана отдельно

**Не трогаем:** `CRM Sync`, GitHub workflow, скрипт sync, колонки A–Q.

## 1. Какие листы создать (и в каком порядке)

Создайте вкладки **слева направо** (первый лист = главный):

| № | Лист | Видимость | Назначение |
|---|------|-----------|------------|
| 1 | **`Dashboard`** | главный | визуальная MM как на скрине |
| 2 | **`Overdue`** | видимый | календарь % по дням месяца |
| 3 | **`_Calc`** | скрыть позже | QUERY / ARRAYFORMULA (мозг) |
| 4 | **`Config`** | скрыть позже | менеджеры, планы, константы |
| 5 | **`Leads`** | скрыть позже | лиды/traffic (ручной ввод до авто-экспорта) |
| 6 | **`Data`** | скрыть позже | IMPORTRANGE из CRM Sync |
| 7 | **`Info`** | опционально | время последнего sync |

**Порядок вкладок:** перетащите `Dashboard` на первое место.

## 2. Лист `Data` — подключение CRM Sync

### 2.1. IMPORTRANGE (единственная связь с CRM Sync)

Ячейка **`Data!A1`**:

```
=IMPORTRANGE("ВАШ_ID_CRM_SYNC"; "Sync!A:Q")
```

1. Замените `ВАШ_ID_CRM_SYNC` на ID из URL таблицы CRM Sync
2. Enter → **«Разрешить доступ»**
3. Дождитесь загрузки строк

### 2.2. Вспомогательные колонки (только на стороне MM, sync не затрагивают)

Добавьте **справа от Q** (начиная с R). Эти формулы ставятся в **строку 2**, протягиваются вниз или через ARRAYFORMULA.

**`Data!R1`** — заголовок: `Месяц_ключ`  
**`Data!R2`**:

```
=ARRAYFORMULA(IF(A2:A="";"";TEXT(A2:A;"YYYY-MM")))
```

**`Data!S1`** — `Текущий_месяц`  
**`Data!S2`**:

```
=ARRAYFORMULA(IF(A2:A="";"";R2:R=TEXT(TODAY();"YYYY-MM")))
```

**`Data!T1`** — `Вчера`  
**`Data!T2`**:

```
=ARRAYFORMULA(IF(A2:A="";"";INT(A2:A)=TODAY()-1))
```

**`Data!U1`** — `Сегодня`  
**`Data!U2`**:

```
=ARRAYFORMULA(IF(A2:A="";"";INT(A2:A)=TODAY()))
```

**`Data!V1`** — `Категория`  
**`Data!V2`**:

```
=ARRAYFORMULA(IF(B2:B="";"";
  IF(B2:B="Возврат";"return";
  IF(LEFT(B2:B;5)="Отказ";"refusal";
  IF(OR(B2:B="Апсэйл";LEFT(B2:B;13)="Доплата Апсэйл");"upsell";
  IF(LEFT(B2:B;7)="Доплата";"topup";
  IF(OR(B2:B="Новая";B2:B="ББ";B2:B="Рассылка");"new";"other")))))))
```

**`Data!W1`** — `Подтип_новой` (для колонок ББ / Проба / Скидка на Dashboard)  
**`Data!W2`**:

```
=ARRAYFORMULA(IF(V2:V<>"new";"";
  IF(B2:B="ББ";"bb";
  IF(B2:B="Рассылка";"mailing";
  IF(REGEXMATCH(L2:L;"(?i)проб");"trial";
  IF(REGEXMATCH(L2:L;"(?i)скид");"discount";"new"))))))
```

> **Проба / Скидка:** в CRM нет отдельных типов сделок — маппим по **курсу (кол. L)** или вручную правим `Config!DealMap`. Если в вашей MM они = отдельные deal types, замените W2 на `=B2:B`.

> **Важно:** лиды **не входят** в 17 колонок sync. Пока вводите вручную (как раньше в MM) или позже добавите отдельный экспорт из CRM.

| A | B | C |
|---|---|---|
| Менеджер (полное имя) | Лиды_месяц | Лиды_сегодня |

**`A2:A10`** = `Config!B2:B10`  
**`B2:B10`** — ручной ввод (или формула позже)  
**`C2:C10`** — ручной ввод «лиды сегодня»

Для **Итого** лидов месяца: **`B11`** = `=SUM(B2:B10)`

## 5. Лист `_Calc` — расчётная матрица (QUERY)

Строка **1** — заголовки. Данные с **строки 2**, по одной строке на менеджера.

### 5.1. Список менеджеров

**`_Calc!A2`**:

```
=ARRAYFORMULA(Config!B2:B10)
```

**`_Calc!B2`** (короткие имена):

```
=ARRAYFORMULA(Config!A2:A10)
```

### 5.2. Формулы в `_Calc` (строка 2, протянуть до строки 10)

Подставьте `$A2` как имя менеджера. Ниже — для **строки 2** (Денис):

| Кол | KPI | Формула |
|-----|-----|---------|
| C | Лиды месяц | `=IFERROR(VLOOKUP($A2;Leads!A:B;2;FALSE);0)` |
| D | Сделки вчера | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$T:$T;TRUE;Data!$B:$B;"<>")` |
| E | Сделки сегодня | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$U:$U;TRUE;Data!$B:$B;"<>")` |
| F | База (мес) | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$O:$O;"Базовый")` |
| G | VIP (мес) | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$O:$O;"VIP")` |
| H | Новые (мес) | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$W:$W;"new")` |
| I | ББ (мес) | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$W:$W;"bb")` |
| J | Проба (мес) | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$W:$W;"trial")` |
| K | Скидка (мес) | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$W:$W;"discount")` |
| L | Рассылка (мес) | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$W:$W;"mailing")` |
| M | Апсейл (мес) | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$V:$V;"upsell")` |
| N | Всего сделок мес | `=COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$B:$B;"<>";Data!$V:$V;"<>return")` |
| O | CV1 | `=IF(C2=0;"";N2/C2)` |
| P | Ср. чек сумма | `=IF(N2=0;"";ROUND(SUMIFS(Data!$E:$E;Data!$Q:$Q;$A2;Data!$S:$S;TRUE)/N2;0))` |
| Q | Ср. чек бюджет | `=IFERROR(ROUND(SUMIFS(Data!$F:$F;Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$F:$F;">0")/COUNTIFS(Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$F:$F;">0");0);"")` |
| R | Выручка вчера | `=SUMIFS(Data!$E:$E;Data!$Q:$Q;$A2;Data!$T:$T;TRUE)` |
| S | Выручка сегодня | `=SUMIFS(Data!$E:$E;Data!$Q:$Q;$A2;Data!$U:$U;TRUE)` |
| T | Выручка новые | `=SUMIFS(Data!$E:$E;Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$V:$V;"new")` |
| U | Выручка доплаты | `=SUMIFS(Data!$E:$E;Data!$Q:$Q;$A2;Data!$S:$S;TRUE;Data!$V:$V;"topup")` |
| V | Выручка всего мес | `=SUMIFS(Data!$E:$E;Data!$Q:$Q;$A2;Data!$S:$S;TRUE)` |
| W | Выручка прошл. мес | `=SUMIFS(Data!$E:$E;Data!$Q:$Q;$A2;Data!$A:$A;">="&Config!$E$4;Data!$A:$A;"<="&Config!$E$5)` |
| X | % выполнения плана | `=IFERROR(V2/VLOOKUP($A2;Config!H:I;2;FALSE);0)` |
| Y | Вознаграждение | см. §5.3 |

**Строка 11 (Итого):** `=SUM(C2:C10)` и аналогично для числовых колонок.  
Для **O (CV1)** в итого: `=IF(C11=0;"";N11/C11)`.  
Для **P, Q** — пересчёт от итогов, не SUM средних.

### 5.3. Вознаграждение (как в CRM salary)

**`_Calc!Y2`**:

```
=LET(
  rev; V2;
  base; Config!$F$1;
  pct; ROUND(rev * Config!$F$2; 0);
  bonus; IF(rev>=Config!$F$5; Config!$F$6; IF(rev>=Config!$F$3; Config!$F$4; 0));
  base + pct + bonus
)
```

> Ночные смены и ручные бонусы в sync не выгружаются — при необходимости добавьте колонку `Bonus` на `Config` и прибавьте к формуле.

### 5.4. QUERY-сводки (опционально, блок ниже на `_Calc`)

**Traffic по источникам (месяц)** — `_Calc!AA1`:

```
=QUERY(Data!A:W;
  "SELECT J, COUNT(A), SUM(E)
   WHERE S = TRUE AND J IS NOT NULL AND J <> ''
   GROUP BY J
   ORDER BY SUM(E) DESC
   LABEL J 'Traffic', COUNT(A) 'Сделки', SUM(E) 'Выручка'";
  1)
```

**Leaderboard (топ по выручке месяца)** — `_Calc!AE1`:

```
=QUERY({_Calc!B2:B10\_Calc!V2:V10\_Calc!N2:N10};
  "SELECT Col1, Col2, Col3
   WHERE Col2 IS NOT NULL
   ORDER BY Col2 DESC
   LABEL Col1 'Менеджер', Col2 'Выручка', Col3 'Сделки'";
  1)
```

> В русской локали элементы массива в `{}` разделяются **обратным слэшем `\`**.  
> В английской — **запятой `,`**: `{_Calc!B2:B10,_Calc!V2:V10,_Calc!N2:N10}`.

| Блок | Ячейки заголовка | Цвет заливки |
|------|------------------|--------------|
| Лиды | D2:D3 | `#F4CCCC` (розовый) |
| Сделки с трафика | F2:U2 | `#D9EAD3` (светло-зелёный) |
| CV1 | U3 | `#FCE5CD` (оранжевый) |

### 6.4. График «Вознаграждения»

1. Диапазон: `Dashboard!B4:B12` (имена) + `=_Calc!Y4:Y12` (вознаграждение)
2. **Вставка → Диаграмма → Столбчатая**
3. Заголовок: **Вознаграждения**
4. Подписи данных: включить (значения на столбцах)
5. Фон области диаграммы: `#E2F0D9`

## 7. Лист `Overdue` — календарь % по дням

Повторяет второй скрин: менеджеры × дни 1…31.

### 7.1. Структура

- **`A1`** — название месяца: `=TEXT(TODAY();"MMMM")` (или ручной «Февраль»)
- **`B1:AF1`** — числа `1` … `31` (формула в B1: `=COLUMN(B1)-1`, протянуть)
- **`A2:A10`** — `=Config!A2:A10` (короткие имена)
- **`B2:AF10`** — % выполнения **дневного** плана

### 7.2. Формула дневного % (ячейка `B2`)

```
=LET(
  d; DATE(YEAR(Config!$E$1); MONTH(Config!$E$1); B$1);
  planDay; IFERROR(VLOOKUP(Config!$B2; Config!H:I; 2; FALSE) / DAY(EOMONTH(d;0)); 0);
  revDay; SUMIFS(Data!$E:$E; Data!$Q:$Q; Config!$B2; Data!$A:$A; d);
  IF(planDay=0; ""; revDay/planDay)
)
```

Протяните:
- вправо до AF (дни),
- вниз до строки 10 (менеджеры).

### 7.3. Условное форматирование

- Если значение **≤ 1%** (или пустой день без продаж) → заливка `#F4CCCC`
- Заголовок и имена → `#FCE5CD`

> Это **% выполнения дневного плана**, не просрочки подписок. Live-просрочки клиентов в sync не выгружаются — они остаются в CRM (Subscriptions). Если нужен отдельный блок просрочек — позже добавим второй экспорт, **не трогая** текущий Sync.

## 9. IMPORTRANGE / QUERY — шпаргалка

| Задача | Функция | Где |
|--------|---------|-----|
| Сырые продажи | `IMPORTRANGE(id;"Sync!A:Q")` | `Data!A1` |
| Время sync | `IMPORTRANGE(id;"_Meta!B2")` | `Info!B2` |
| Сводка traffic | `QUERY(Data!...)` | `_Calc` |
| Leaderboard | `QUERY` + сортировка | `_Calc` |
| KPI по менеджеру | `COUNTIFS` / `SUMIFS` | `_Calc` |
| % от total | Dashboard | деление на `$N` (всего сделок) |
| Массив по всем строкам | `ARRAYFORMULA` | `Data!R:W` |

**IMPORTRANGE не realtime:** после nightly sync нажмите **F5** в MM Dashboard (или подождите авто-обновление ~до минуты).

## 11. Что автоматизировано vs ручной ввод

| Блок | Источник | Авто? |
|------|----------|-------|
| Сделки / выручка / типы / тарифы | `Data` (sync) | ✅ |
| Средний чек | `Data` E, F | ✅ |
| CV1 | сделки / лиды | ⚠️ лиды ручные (`Leads`) |
| Планы месяца | `Config!I` | ✋ ручной |
| Overdue-календарь (% плана) | `Data` + план | ✅ после плана |
| Вознаграждение | формула salary | ✅ (без ночных/ручных бонусов) |
| Просрочки подписок | CRM Subscriptions | ❌ не в sync |
| Traffic-лиды из CRM | Firestore traffic | ❌ пока не в sync |

## 12. Типичные проблемы

| Симптом | Причина | Решение |
|---------|---------|---------|
| Везде 0 | Имена менеджеров не совпадают | Сверить `Data!Q` и `Config!B` |
| #REF! на Data | Нет доступа IMPORTRANGE | «Разрешить доступ» |
| #N/A в QUERY | Пустой Data | Дождаться nightly sync |
| CV1 пустой | Лиды = 0 | Заполнить `Leads!B` |
| Проба/Скидка всегда 0 | Нет в курсе | Править `Data!W2` или deal map |
| Старые данные | Кэш IMPORTRANGE | F5 / Ctrl+Shift+F9 |

## 13. Безопасность sync (напоминание)

**Не делайте:**
- не переименовывайте `Sync` / `_Meta` в CRM Sync
- не вставляйте формулы в CRM Sync (только MM Dashboard)
- не меняйте порядок колонок A–Q в sync-скрипте без обновления MM

**Можно:**
- скрывать служебные листы MM
- добавлять колонки R+ только в MM `Data`
- менять оформление `Dashboard` как угодно
