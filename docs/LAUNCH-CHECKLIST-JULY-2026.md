# Чек-лист запуска CRM — 1 июля 2026

## Автоматически (без ваших действий)

С **1 июля 00:00** по локальному времени браузера CRM переключается на **июльский** период:

| Что | Июнь | Июль |
|---|---|---|
| **Выручка месяца** (главная, KPI) | только оплаты с `paymentDate` в июне | только оплаты с `paymentDate` в июле |
| **Зарплата — текущий месяц** | — | только июльские оплаты + июльские смены/бонусы |
| **Зарплата — архив** | — | блок **«Архив · Июнь 2026»** (полный месяц) |
| **Лидер месяца / рейтинг** | — | считается по июльским оплатам |
| **История оплат** | все оплаты | все оплаты (без изменений) |

**Важно:** месяц определяется по полю **«Дата оплаты»** в форме, не по дате сохранения.

## Перед запуском (сегодня)

### 1. Продакшен
- [ ] На GitHub последний коммит задеплоен: **Deploy Production** → зелёная галочка
- [ ] Открыть https://crm-school-v2.web.app — сайт грузится
- [ ] Войти под admin и под 1–2 менеджерами — логин работает

- [ ] У каждого менеджера есть логин в CRM (Firebase Auth)
- [ ] Роль `manager`, правильный `managerId`

- [ ] Обновить GitHub Secret `MANAGER_TT_SPREADSHEETS_JSON` — все 9 менеджеров

### 4. График на 1 июля
- [ ] **Управление → График** — выходные/замены на июль настроены

## Утром 1 июля — быстрая проверка (5 мин)

### Admin
- [ ] **Главная → Выручка месяца** = 0 или только сегодняшние июльские оплаты (не июнь!)
- [ ] **Leaderboard** — без июньских сумм (если ещё нет июльских оплат)
- [ ] **Зарплата** — заголовок **«Июль 2026»**, архив **«Июнь 2026»**

### Менеджер
- [ ] **Мои KPI** — выручка за июль (с 1-го)
- [ ] **Зарплата** — только вкладка «Зарплата», без «Ночные смены» / «Бонусы»
- [ ] **Новая оплата** — тестовая запись с датой **01.07.2026** → появилась в KPI

## Правила для команды с 1 июля

1. **Новые оплаты** — только **CRM → Новая оплата**
2. **Июньские подписчики** (перенос базы) — «Продолжить как старый клиент»
3. **Повторные доплаты** старых — «Доплата (старый подписчик)»
4. **Дата оплаты** — ставить **реальную** дату платежа:
   - июльская продажа → дата в июле → идёт в июльскую ЗП
   - перенос июньской оплаты → дата в июне → идёт в **архив июня**, не в июльскую ЗП
5. **Google Таблицы** — июнь как архив; июльские новые строки дублировать в CRM не нужно

## Если что-то не так 1 июля

| Симптом | Что проверить |
|---|---|
| В KPI июньские суммы | **Дата оплаты** у записи — возможно стоит июнь |
| ЗП не обнулилась | Открыть «Зарплата» — должно быть «Июль 2026»; июнь в архиве |
| Оплата не в ТТ | Actions sync; `MANAGER_TT_SPREADSHEETS_JSON`; вкладка **ТТ** |
| Менеджер видит чужие данные | Роль и managerId аккаунта |
| Сайт старая версия | Deploy Production + Ctrl+F5 |
